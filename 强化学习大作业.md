# 强化学习大作业：众包任务推荐

## 背景知识

众包模型旨在从相对开放、快速变化的参与者群体中获得服务。众包通常通过互联网吸引参与者，将工作任务划分并分配给参与者，并在任务完成后聚合结果。

非营利机构可以使用众包模型开发公共服务，例如维基百科。此外，当前也存在一些商业系统。例如，Amazon 推出了 Amazon MTurk 系统，提供收费的众包服务，以吸引人工解决计算机难以处理的问题。

在 Amazon MTurk 上，请求者首先发布任务（例如标注图片），并给出相应报酬（例如完成一张图片的付费金额）。当参与者进入系统时，平台会向该参与者展示可获取的任务列表；参与者可以查看任务的详细信息，包括标题、描述、创建时间、结束时间等。最后，参与者选择任务，并决定是否完成。

## 问题描述

在众包系统中，当参与者进入平台时，系统需要解决任务推荐或任务排序问题，即如何根据参与者的特征和任务状态选择要展示的任务。

为简化问题，假设系统每次只向参与者推荐一个任务。由于系统是盈利性的，众包系统需要同时满足参与者和请求者两方的利益：

1. 参与者可以找到更多相关且感兴趣的任务，从而赚取更多报酬。
2. 请求者发布的任务可以得到更多、更高质量的回答。

与此同时，由于参与者和请求者都是动态变化的，系统在分配任务时应能够考虑并处理这种动态性。

## 具体任务

1. 如何将强化学习技术应用到众包任务推荐中，并最大化**参与者**的利益？
2. 如何将强化学习技术应用到众包任务推荐中，并最大化**请求者**的利益？

## 实验要求

给出具体的实验流程和设计细节，并给出最终的实验结果。

**要求使用 DQN 系列模型解决上述问题。**

## 实验数据

附件包括以下数据与参考文件：

- `sample_read_data.py`：用于读取数据的参考代码。
- `worker_quality.csv`：包含 worker 的 `quality` 属性。
- `project_list.csv`：包含 `project_id` 和 `project_answer_num`。
- `project/`：包含 project 相关信息。
- `entry/`：包含 worker 相关信息。

为简化问题，可直接参考 `sample_read_data.py` 的内容读取数据。获取数据后，需要自行划分训练集、验证集和测试集，并在实验报告中说明划分细节。

## 提示

1. 如何分别针对参与者和请求者定义状态、动作、奖励等？
2. 如何定义当前状态和下一步状态？
3. 如何构造 Q 函数？
4. 如何分析数据并构造特征？
5. 如何根据任务创建时间、到期时间以及 worker 到达时间进行排序，从而模拟 worker 的兴趣和完成情况？
